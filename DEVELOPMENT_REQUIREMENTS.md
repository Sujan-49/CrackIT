# CrackIT by SJ DEVS - Development Requirements

CrackIT is a functional educational platform, not a landing page or UI showcase.

Priority order:

1. Working backend
2. Database
3. Authentication
4. APIs
5. Functional features
6. Data persistence
7. Responsive UI
8. Animations
9. Visual effects

Do not build fake dashboards, fake analytics, fake streak systems, fake leaderboards, fake statistics, fake notifications, fake user data, or placeholder functionality.

If a feature cannot be implemented yet, leave it visibly unfinished rather than simulating it.

Version 1 should focus on:

- Authentication
- User profiles
- DSA tracking
- MCQ engine
- Bookmarks
- Notes
- Roadmap progress
- Company preparation
- Persistent progress

Reference Platform Notice:

CrackIT is designed as a learning and reference resource. While we provide curated technical content, interview topics, and career guidance, users should not rely entirely on this platform. Always verify information, build practical projects, explore official documentation, and use this website as a supplementary learning resource.
